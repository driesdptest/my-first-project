# Custom Numpad Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the native keyboard on the exercise screen with a custom in-UI numeric keypad so mobile users never see the system keyboard.

**Architecture:** The answer `<input>` becomes `readonly` (no native keyboard). A 3×4 grid of buttons below the feedback area handles digit input, clearing (Wis), and submission (✅). Event delegation on the `.numpad` container routes all taps. `btnControleren` is removed entirely.

**Tech Stack:** Plain HTML5, CSS Grid, vanilla JS ES modules — no dependencies.

- [ ] **Step 3: Replace btnControleren listeners with numpad handler**

Find these two lines (around line 200–201):

```js
btnControleren.addEventListener('click', checkAnswer);
answerInput.addEventListener('keydown', e => { if (e.key === 'Enter') checkAnswer(); });
```

Replace them with:

```js
document.querySelector('.numpad').addEventListener('click', e => {
  const btn = e.target.closest('.numpad-btn');
  if (!btn || btn.disabled) return;
  const val = btn.dataset.val;
  if (val === 'wis') {
    answerInput.value = '';
  } else if (btn.classList.contains('numpad-ok')) {
    checkAnswer();
  } else {
    if (answerInput.value.length < 4) {
      answerInput.value += val;
    }
  }
});
```
